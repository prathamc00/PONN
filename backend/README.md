# CRISMATECH PHP Backend — Hostinger Deployment Guide

## Step 1: Set Up MySQL on Hostinger

1. Go to **Hostinger hPanel → Databases → MySQL Databases**
2. Create a new database (e.g., `u123456_crismatech`)
3. Create a database user and assign it to the database
4. Click **phpMyAdmin** to open the database manager
5. Select your database → click **Import** → upload `schema.sql`

---

## Step 2: Configure Environment Variables

1. **Copy** `.env.example` to `.env`
2. Fill in all values:

```
DB_HOST=127.0.0.1
DB_NAME=u123456_crismatech    ← your Hostinger DB name
DB_USER=u123456_myuser        ← your Hostinger DB user
DB_PASS=your_db_password

JWT_SECRET=any_very_long_random_string_here

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_gmail_app_password    ← Gmail App Password (not your login password)

FRONTEND_URL=https://yourdomain.com,https://www.yourdomain.com
RESET_PASSWORD_URL=https://yourdomain.com/reset-password
```

---

## Step 3: Install PHP Dependencies (via Hostinger SSH or File Manager)

Hostinger Premium supports SSH. Connect via SSH and run:

```bash
cd public_html
composer install --no-dev --optimize-autoloader
```

If you don't have SSH, you can:
1. Install Composer locally on your PC
2. Run `composer install` in this `php-backend/` folder locally
3. Upload the generated `vendor/` folder to `public_html/`

---

## Step 4: Upload Files to Hostinger

Upload the **contents** of the `php-backend/` folder to `public_html/`:

```
public_html/
├── .htaccess
├── index.php
├── .env               ← your filled-in .env (not .env.example!)
├── config/
├── middleware/
├── helpers/
├── modules/
├── vendor/            ← generated by composer install
└── uploads/           ← create this folder manually (chmod 755)
```

> **Important:** Do NOT upload `composer.json` or `.env.example` publicly accessible without protection. The `.htaccess` already blocks access to these files.

---

## Step 5: Upload the Frontend

1. Run `npm run build` inside the `frontend/` folder
2. Upload the contents of `frontend/dist/` to `public_html/` **as well**

The result will be:
```
public_html/
├── index.html         ← React app entry point
├── assets/            ← React compiled JS/CSS
├── .htaccess          ← Routes /api/* to PHP, and non-API to React
├── index.php          ← PHP API router
└── ...
```

---

## Step 6: Update Frontend API URL

In your React frontend, find where the `API_URL` or base URL is set, and change it to:

```
https://yourdomain.com/api
```

Then rebuild the frontend (`npm run build`) and re-upload `dist/` contents.

## Notes

- **Real-time notifications** are replaced with polling. The bell icon works by fetching from `GET /api/notifications` every 10–15 seconds.
- File uploads go to the `uploads/` folder. Make sure this directory exists and is writable.
- PHP version required: **8.0+** (Hostinger supports PHP 8.1/8.2 — select it in hPanel → PHP Configuration)
